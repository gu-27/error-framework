The Framework

Understanding The Error Framework

Why Error?

Error Language Enables Learning. Failure Language Shuts It Down.

Most people frame problems as failures. Failure language triggers shame, defensiveness, and blame — it shuts down learning and growth. Error language comes from research, engineering, and computer science, where error is expected, measurable, and fixable. In signal processing, error is noise in the system you work to minimize. In software, an error is something you debug and iterate on.

The Error Framework borrows this mindset: every problem or negative outcome has contributing error sources. When we name them precisely, we can address them. When we own them honestly, we grow.

The Formative Construct

Problems Are Formed by Multiple Sources — Not Caused by One.

In research methodology, a formative construct is built from its contributing dimensions — the factors don't just measure the construct, they form it. The Error Framework applies this same logic to real-world problems. A negative outcome is not caused by one thing — it is formed by multiple error sources working together.

This matters because it means no single party fully owns the problem. That is not an excuse. It is precision. And precision is what makes growth conversations possible instead of defensive ones.

The Three Error Sources

Every Problem Has Three Contributing Dimensions.

Evaluator Error

The Assessor's Limitations

The person assessing the situation brings their own limitations: bias, incomplete context, preconceived ideas, or lack of experience in the specific domain. Evaluator error does not mean the evaluator is wrong — it means their perspective is shaped by their own constraints, and naming that creates space for honesty.

Situational Error

Factors Outside Anyone's Control

Time pressure, resource constraints, environmental conditions, organizational dynamics, external events. Even in an ideal scenario with capable, well-intentioned people, situations create error. Naming situational factors is not excuse-making — it is context that belongs in the picture.

Actor Error

The Individual's Contribution

The person most directly connected to the outcome has their own contributing error: skill gaps, distraction, poor time management, inexperience, disengagement. This is the error most directly tied to individual growth. It does not define the person — it defines what can be improved.

A Concrete Example

The Missed Deadline.

A student misses a project deadline. At first glance, the story is simple — the student didn't turn it in. But the full picture is more precise.

Breaking It Down

Actor error: poor time management and underestimating the scope of the work.

Situational error: a family emergency during the final week.

Evaluator error: unclear expectations set at the start of the assignment.

All three contributed to the outcome. The student still owns their actor error — but the full picture matters for an honest conversation about what actually needs to change.

How to Use It

A Collaborative Conversation Tool.

Start by naming the problem or negative outcome clearly. Then work through each error source together — what evaluator limitations were present? What situational factors played a role? What actor behaviors contributed? Use the tool below to assign a rough weight to each source. The visualization updates in real time to show how each error source shapes the whole.

A recommended starting move: begin with evaluator error. When the person running the conversation names their own limitations first, it lowers defensiveness and models the kind of honest ownership the conversation is asking for.
